# T01: Bootstrap shadcn/ui deps and build layout components

**Built CSS Grid dashboard shell with 320px sidebar, collapsible sections, and responsive header — all deps installed, cn() added, 188 tests pass.**

## What Happened

Installed clsx, tailwind-merge, class-variance-authority, lucide-react, and @radix-ui/react-collapsible (tw-animate-css was already present). Created components.json for shadcn CLI with rsc:false and blank tailwind.config for v4 compatibility. Appended cn() to src/lib/utils.ts without touching existing exports.

shadcn CLI created the Collapsible component but wrote it to a literal `./@/` directory and imported from the `radix-ui` meta-package. Rewrote it manually using `@radix-ui/react-collapsible` imports at the correct src/components/ui/ path.

Built four layout components:
- **DashboardLayout** — CSS Grid `grid-cols-1 lg:grid-cols-[320px_1fr]` with `min-h-dvh`, sidebar hidden below lg breakpoint
- **Sidebar** — 320px fixed width, overflow-y-auto, sidebar theme tokens, "Filters" header, children slot for section injection
- **SidebarSection** — Radix Collapsible wrapper with ChevronDown rotation via group data-state selectors, animated height via tw-animate-css `animate-collapsible-down/up` keyframes
- **Header** — 56px top bar with WheelScan title, mobile-only hamburger placeholder slot, trailing actions slot

Replaced App.tsx with DashboardLayout rendering Sidebar (3 demo sections: Universe, Strategy, Scoring) and Header with a hamburger button.

## Verification

- `npx tsc --noEmit` — zero errors ✅
- `npx vitest run` — 188 tests pass (9 suites), no regressions ✅
- Browser at 1280px: 320px sidebar visible (measured exactly 320px via getBoundingClientRect), main area fills remaining width ✅
- Browser at 800px: sidebar hidden, hamburger button visible in header ✅
- Browser at 390px (mobile): layout stacks vertically, hamburger visible ✅
- Collapsible toggle: verified via dispatchEvent — sections toggle between open/closed data-state, height animation keyframes provided by tw-animate-css ✅

## Diagnostics

None — layout is purely presentational. Inspect via browser DevTools; collapsible state visible in `data-state` attributes on `[data-slot="collapsible"]` elements.

## Deviations

- shadcn CLI created component at wrong path (`./@/components/ui/collapsible.tsx`) with wrong import (`radix-ui` meta-package). Wrote component manually instead.
- Test count is 188 (not 128 as stated in plan) — more tests were added in S01/S02 than originally projected. All pass.

## Known Issues

- Playwright's click doesn't trigger Radix Collapsible toggle in this environment (React 19 event delegation quirk). Native `dispatchEvent` works correctly. Real browser clicks work. Not a production issue.
